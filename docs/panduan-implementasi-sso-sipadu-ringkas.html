Quick Execution Guide

Checklist Implementasi SSO SIPADU untuk Programmer

Versi singkat ini dibuat agar tim bisa langsung eksekusi. Fokusnya task, to-do list, dan output yang harus selesai di tiap aplikasi surrounding.

Dokumen internal • Ringkasan implementasi • 8 April 2026

1. Target Implementasi

  • User login di SIPADU, lalu masuk ke aplikasi surrounding tanpa input password lagi.
  • Pencocokan user dilakukan dengan employee_id.
  • Role aplikasi tetap memakai data lokal aplikasi masing-masing.

2. Data Minimal yang Harus Ada

Di SIPADU

employee_id, name, email

Di Aplikasi Surrounding

employee_id, name, email, is_active

Catatan: untuk matching user, gunakan employee_id sebagai kunci utama. Email dan nama hanya data pendukung.

3. Task Step by Step

Step 1. Tambah kolom employee_id

Tujuan: supaya user di aplikasi surrounding bisa dicocokkan dengan user SIPADU.

  • Tambah kolom employee_id di tabel users.
  • Tambahkan unique index jika satu user satu ID karyawan.
  • Isi data existing user dengan employee_id yang benar.

Step 2. Tambah konfigurasi SSO di aplikasi

Tujuan: aplikasi tahu siapa issuer dan secret yang dipakai untuk validasi token.

  • Tambahkan SSO_ENABLED di .env.
  • Tambahkan SSO_ISSUER, SSO_AUDIENCE, dan SSO_SHARED_SECRET.
  • Mapping config ke config/services.php atau file config khusus.
SSO_ENABLED=true
SSO_ISSUER=sipadu
SSO_AUDIENCE=dmtl-audit
SSO_SHARED_SECRET=secret-aplikasi

Step 3. Buat route penerima SSO

Tujuan: menyediakan endpoint yang akan dipanggil SIPADU saat user membuka aplikasi.

  • Tambahkan route /sso/login.
  • Route menerima parameter sso_token.
Route::get('/sso/login', [SsoLoginController::class, 'store'])
    ->name('sso.login');

Step 4. Buat service validasi token

Tujuan: memastikan token hanya diterima jika valid dan memang berasal dari SIPADU.

  • Cek signature token.
  • Cek iss harus SIPADU.
  • Cek aud harus sesuai aplikasi.
  • Cek token belum expired.
<?php

namespace App\Services;

use Illuminate\Support\Arr;
use Illuminate\Support\Carbon;

class SsoTokenService
{
    public function decode(string $token): array
    {
        [$encodedPayload, $signature] = explode('.', $token, 2);

        $expectedSignature = hash_hmac(
            'sha256',
            $encodedPayload,
            config('services.sso.shared_secret')
        );

        if (! hash_equals($expectedSignature, $signature)) {
            abort(403, 'Signature token tidak valid.');
        }

        $payload = json_decode(base64_decode($encodedPayload), true);

        if (! is_array($payload)) {
            abort(403, 'Payload token tidak valid.');
        }

        if (Arr::get($payload, 'iss') !== config('services.sso.issuer')) {
            abort(403, 'Issuer token tidak valid.');
        }

        if (Arr::get($payload, 'aud') !== config('services.sso.audience')) {
            abort(403, 'Audience token tidak valid.');
        }

        if (Carbon::createFromTimestamp(Arr::get($payload, 'exp'))->isPast()) {
            abort(403, 'Token sudah expired.');
        }

        return $payload;
    }
}

Step 5. Buat controller login SSO

Tujuan: setelah token valid, aplikasi login-kan user secara lokal.

  • Ambil employee_id dari token.
  • Cari user lokal berdasarkan employee_id.
  • Pastikan user aktif.
  • Jalankan Auth::login($user).
  • Redirect ke halaman utama aplikasi.
<?php

namespace App\Http\Controllers;

use App\Models\User;
use App\Services\SsoTokenService;
use Illuminate\Http\RedirectResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Auth;

class SsoLoginController extends Controller
{
    public function store(Request $request, SsoTokenService $ssoTokenService): RedirectResponse
    {
        abort_unless(config('services.sso.enabled'), 404);

        $token = (string) $request->query('sso_token');
        abort_if($token === '', 403, 'Token SSO tidak ditemukan.');

        $payload = $ssoTokenService->decode($token);
        $employeeId = data_get($payload, 'user.employee_id');

        $user = User::query()
            ->where('employee_id', $employeeId)
            ->where('is_active', true)
            ->first();

        abort_if(! $user, 403, 'User tidak terdaftar atau tidak aktif.');

        Auth::login($user);
        $request->session()->regenerate();

        return redirect()->intended('/dashboard');
    }
}
Catatan untuk Filament: jika aplikasi memakai panel Filament, redirect akhir biasanya diarahkan ke /admin atau URL panel yang dipakai. Contoh: return redirect()->intended('/admin');

Step 6. Tambah replay protection

Tujuan: token yang sama tidak bisa dipakai dua kali.

  • Simpan nilai jti token yang sudah dipakai.
  • Jika jti sudah ada, tolak login.

Step 7. Tambah audit log

Tujuan: semua login dari SIPADU bisa ditelusuri.

  • Simpan user_id atau employee_id.
  • Simpan token_jti.
  • Simpan waktu login, IP address, dan user agent.

4. To-Do Checklist Final per Aplikasi

  • Kolom employee_id sudah ada di tabel user.
  • Data user existing sudah diisi employee_id.
  • Config SSO di .env sudah tersedia.
  • Route /sso/login sudah aktif.
  • Service validasi token sudah dibuat.
  • Controller auto-login sudah dibuat.
  • Matching user memakai employee_id.
  • Replay protection dengan jti sudah aktif.
  • Audit log login SSO sudah tercatat.
  • Testing happy path dan invalid token sudah selesai.

5. Testing yang Wajib Dilakukan

Happy Path

  • User valid bisa login otomatis.
  • User masuk ke halaman aplikasi setelah SSO.

Negative Test

  • User tidak ditemukan harus ditolak.
  • Token expired harus ditolak.
  • Audience salah harus ditolak.
  • Signature salah harus ditolak.
  • Token replay harus ditolak.

6. Pembagian Task Tim

Tim SIPADU

  • Siapkan launch URL aplikasi.
  • Siapkan audience aplikasi.
  • Siapkan shared secret per aplikasi.

Tim Surrounding App

  • Update tabel user.
  • Buat endpoint SSO.
  • Buat validator token dan auto-login.
Rekomendasi eksekusi: pilih satu aplikasi pilot dulu, jadikan template, lalu copy pola yang sama ke aplikasi surrounding berikutnya.
